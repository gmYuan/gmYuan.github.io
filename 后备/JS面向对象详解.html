JS面向对象详解

一 前言

JS里面向对象相关的知识点很重要，所以最近对这一部分内容做了学习和小结。

二 对象属性

S1 对象的本质是: 一组 无序key-value对 数据集合 / 属性的集合
  在内存中用heap表示

S2 对象的属性和特性
从上文可知，对象就是一个个 无序的属性 的集合，但是，即使是在同一个对象里，它里面的属性之间却也可能有很大的差异性
比如: 有些属性值我们是可以改变的，有些却不可以（甚至都无法删除），这是为什么呢？

A1 首先 要明确对象在内存中的存储方式:
  (1)变量名: 值（变量值是 基本类型） / 内存地址addr（变量值是 引用类型）
  (2)引用类型（key-value对）是以字典形式存储的，属性名: 属性值对象（称为属性描述对象）

也就是说， var example = {foo: 5} , example.foo的访问过程是:
查找到example标识符—> 指向一个内存地址addr1—> 在addr1里查找到一个对象X—> X里查找到一个标识符foo—> foo又指向一个对象Y—> 在对象 Y里有一个[[value]]特性——> [[value]]存储的值是5—> 输出example.foo = 5
还不清楚的 可参考阮一峰的 这篇文章:
JavaScript 的 this 原理

A2 由A1可知，对象中的属性名，指向的属性值都是一个 属性描述对象;
  这个属性描述对象里也有一些属性，为了和“外层属性”区别，高程里称其为 特性。
  换言之，就是特性决定了对象里属性的表现（是否可访问、可枚举等）

A3 根据所含特性的不同，我们可以把对象中的属性分为两种: 数据属性和访问器属性
数据属性
  含有 [[ configurable]]、[[enumerable]]、[[writable]]、[[value]]特性的属性，称为 数据属性
访问器属性
  含有 [[ configurable]]、[[enumerable]]、[[get]]、[[set]]特性的属性，称为 访问器属性

关于这6种特性的具体含义，可参考高程P138~144，或者阮一峰的教程:
属性描述对象

A4 因为特性是用来让JS引擎描述属性行为的，所以我们无法直接访问和修改特性，但是
  可以用ES5提供的 Obdect.defineProperty() 和 Object.defineProperties() 设置对象的属性和属性特性;
  可以通过 Object.getOwnPropertyDescriptor() 获取属性描述对象
它们的用法 同样可见A3步骤参考教程

A2 判断实例和原型的关系:
