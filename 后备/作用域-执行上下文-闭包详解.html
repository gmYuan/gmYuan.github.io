作用域+执行上下文+闭包详解

一 前言

在学习闭包的过程中，学习到了闭包和执行上下文的相关知识，觉得收获很大。

二 作用域

1 什么是作用域
  可以把作用域看做是一个抽象的规则/定义，它规定了查找变量的规则

2 作用域类型
  作用域可以分为词法作用域（即静态作用域）和动态作用域
  JS采用的是词法作用域

3 词法作用域
  词法作用域，意味着 作用域在书写代码时在函数定义时就确定了；
  这句话是什么意思？

  S1我们可以把词法作用域看做是一个密闭空间，在这个空间里存储了变量的相关信息
  S2 在ES5之前，只有全局作用域和函数作用域。换言之，在写代码时每创建一个函数，我们就单独建立了一个封闭空间
  S3 根据书写代码时的 函数间嵌套，内部房间可以访问外部房间内的信息，但反之则不行
    （即 词法作用域的访问规则是： 只能根据书写位置，从内至外查询变量）
  S4 以上就是所谓 在函数定义时就确定作用域 的含义

需要注意的是：
  （1）查询时遇到第一个匹配的变量字符就会停止向外层继续寻找，所以具有“遮蔽效应”
  （2）词法作用域只会查找一级标识符，如foo.bar中的foo

4 动态作用域
  在调用函数时，作用域才会明确（封闭空间才会形成）

6 创建作用域的作用
  （1）可以封闭/隐藏不想暴露给外层（全局）的 变量和函数
  （2）可以创建独特的命名空间（对象），避免同名变量的覆盖/冲突

7 块作用域
JS中现在也有一些可以创建块作用域的方法：
  (1)with语句
  (2)try..catch中的catch分句
  (3)let/const
关于let关键字相关内容，以后会单独另写一篇博文（应该是属于ES6的学习专题了）

这里需要注意的是：

3.2 执行上下文

1 含义
  S1 JS不是一句一句执行代码，而是以段为单位执行代码的
  S2所谓“代码段”，就是一段文本形式的代码
  S3 JS遇到以下3种“代码段”调用时，会 预先在程序内部 做一些处理和准备工作(形成执行上下文)
    (1) 全局代码
    (2) 函数代码
    (3) eval代码
  S4 这里的准备工作，就是所谓的 “执行上下文”

每个执行上下文都做了3件准备工作，以明确3个属性值，这3个属性分别是：
  （1）变量对象(Variable object，VO)
  （2）作用域链(Scope chain)
  （3）this

2 变量对象
  S1变量对象存储了在此执行上下文环境中，创建的变量和函数 的相关信息
  S2 全局上下文中的变量对象 就是全局对象，这也可以解释 为什么全局对象可以通过属性访问全局变量
    （1）全局上下的VO中，没有arguments对象

  S3 函数上下文中，用活动对象(activation object, AO) 表示 变量对象（VO）。
    （1）本质上，两者是一个东西：变量对象是抽象概念，活动对象是实际实现
    （2）在函数执行时，AO = VO + function parameters + arguments

S4 执行上下文中的代码会分为两个阶段被处理：
内部准备（引擎分析）阶段
  变量对象AO包括：
    以arguments对象初始化
    函数的所有形参（值为参入的实参）
    函数声明（值为函数地址）， 如果已经存在相同名称的函数属性，则完全替换这个属性
    变量声明（值为undefined），若变量名称 跟已声明的形参/函数相同，则忽略后者声明

4 总结一下以上内容:
S1 书写全局代码:
  （1）执行上下文栈压入全局对象，即 ECStack = [globalContext]
  （2）初始化全局作用域的变量对象 VO及3个属性值
  （3） 函数被创建时，内部就有一个[[scope]]属性，即fn.[[scope]] = [globalContext.VO]

S2 执行函数fn()时，创建当前的函数执行上下文对象，并压入执行上下文栈，即ECStack = [globalContext , fnContext]

S3 fn函数进入执行准备阶段
  （1）复制函数[[scope]]属性 fnContext = { Scope: fn.[[scope]] }
  （2）创建当前函数的 变量对象AO
    建立arguments对象 (全局环境下没有这一步) + 记录形参（值为传入的实参）
    进行函数声明提升，值为函数地址addr + 它会覆盖同名变量（包括形参+变量）
      注意：在if等语句块{}中的函数声明也会被提升
    变量声明提升，值为undefined + 不会覆盖同名变量（已有的话，之后的会被忽略）
      注意：没有用var声明就 直接创建/赋值的变量，不会进行提升

  （3）把当前上下文对象， 压入Scope顶端，形成完整的作用域链

  （4）this初始化为undefined

S4进入实际执行阶段
  进行变量占位值真正赋值/覆盖 + 确定this的值

四 闭包

1 什么是闭包
  S1 理论上的闭包，是指那些 能够访问自由变量的函数
  S2 自由变量：在函数中使用的，但既不是函数参数也不是函数的局部变量 的变量
    说人话就是： 在函数外定义的变量
  S3 所以，闭包 = 函数 + 函数可以访问的自由变量

  S4从实际使用角度，满足以下条件才是闭包:
    （1）函数所属的父级上下文已销毁，但它仍然存在（比如，内部函数从父函数中返回）
    （2）函数在内部引用了自由变量
