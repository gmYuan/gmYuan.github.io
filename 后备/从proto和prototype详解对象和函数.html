从__proto__和prototype详解对象和函数

一 前言

在学JS过程中，之前一直对函数和对象的关系有疑惑。一方面，在JS中 函数也是对象；另一方面，对象却又是通过构造函数实现的实例
那么他们的关系到底是什么?

PS: 关于对象是什么，以及对象是 通过构造函数实现的，可参考
01 深入理解javascript原型和闭包(1-2)

二 前置知识和规则

在具体介绍之前，先介绍一下前置知识，这是理解下文的基础

S1 对象:
A1 对象是 一组key-value 数据(即属性) 的集合;
方法本质上是 属性值是函数的 特殊属性;

S2 每个对象A 都有一个__proto__属性
A1 proto属性的特点: 指向另一个对象B, 我们可以极不准确的认为，B是A的 “父对象”;
A2 proto属性的作用: 通过它，对象A除了可以访问自身定义的属性值(包括特殊属性—方法)，还可以访问B对象的属性;

S3 函数:
A1 函数是 一个有特殊属性(prototype属性) 的特殊对象，
换言之，函数也是对象，可以具有属性，它同时具有proto属性和prototype属性;
函数对象的最大作用是: 它具有独立作用域 + 可以独立封装/执行一段代码片段，具有复用性;

A2 prototype属性的特点: 函数对象F的prototype属性，必然指向一个叫做 F.prototype的对象:
如果这个F.prototype对象 已存在， 那就直接指向它;
如果这个F.prototype对象 还不存在，那就立刻在函数定义执行后生成它;

A3 prototype属性的作用: 当函数被作为特殊的构造函数使用时，可以让新创建出来的 实例对象可以继承 F.prototype对象的属性，
换言之，
它的作用就是 引导 实例对象的proto属性，使其指向F.prototype对象，从而构成原型链

PS: 关于prototype属性，可参考
01 深入理解javascript原型和闭包(3)

三 (构造)函数和对象 的过程详解

S1
A1 首先，浏览器内置了一个对象，叫做Object.prototype, 它是一个对象 + 对象都有proto属性
ES规定了: Object.prototype.__proto = null;

A2 同时，Object.prototype对象 预先规定了 自己的一些属性和方法 (如 toSting方法);

S2
A1 接着，浏览器又实现了另一个对象，叫做Function.prototype,它也是一个对象 + 对象都有proto属性
ES又规定了: Function.prototype.__proto = Object.prototype
也就是说，我们可以简单理解为 Object.prototype对象 是 Function.prototype对象 的父对象;

A2 同时，Function.prototype对象 也预先规定了 自己的一些属性和方法 (如 apply/call方法);

S3
A1 再后，浏览器又又实现了 另2个比较特殊的对象，分别叫做Function函数对象 和 Object函数对象
根据博文第二部分内容可知， 这2个都有一个独有的属性, 就是 prototype属性
所以， Function 函数(对象) / Object函数(对象) 就同时有这两个属性 proto + prototype ;

A2 因为 函数对象F的prototype属性，必然指向一个叫做 F.prototype的对象，所以:
Function.prototype值 = Function.prototype对象 (即 S2步骤 浏览器实现的内置对象),
Object.prototype值 = Object.prototype对象 (即 S1步骤 浏览器实现的内置对象);

A3 同时，ES规定:
Function.proto值 = Function.prototype对象, Object.proto值 = Function.prototype对象,
也就是说，可以理解为: Function.prototype对象 同时是 Object函数对象 和 Function函数对象 的父对象;

S4
A1 有了 Object函数对象 和 Function函数对象，我们就可以构造出自己的 object实例对象 和 function实例对象了;

A2 实例对象的proto属性值 指向的是 构造函数对象的prototype的 值对象,
也就是说，实例对象和 (构造)函数对象的 原型对象之间， 通过 proto属性和prototype属性，建立了联系
这就是原型链

S5
A1 同理， Array等函数(对象)，也都有 proto属性 和 prototype属性，且 proto值 = Function.prototype对象
也就是说，我们可以简单理解为 Function.prototype对象 还构造出了 Array等其他函数对象

综合上面所有内容，一张经典图理解为:

小结一下就是:

  1. 所有对象(包括函数)都有 proto原型属性;
  2. 只有 (构造)函数 才有prototype属性;
  3. 构造函数的 proto原型属性 与 它的prototype属性 不是一回事
    前者是它作为对象时的属性，表示这个函数是谁生的, 或者说是在谁的基础上扩展来的
    后者是它作为一个构造函数时特有的属性，即用它来new一个对象时，这个被new出来的对象是用谁做模版的;
